# 代码规范检查工具

## 功能
- 检查代码上传权限
- 确保没有明确说明的情况下不会进行CI上传
- 确保没有明确说明的情况下不会上传到git
- 提供代码规范指导

## 使用方法

### 检查代码规范
```
@code-standard check
```

### 运行沙箱测试
```
@code-standard test
```

### 允许CI上传
```
@code-standard allow ci
```

### 允许git上传
```
@code-standard allow git
```

### 同时允许CI和git上传
```
@code-standard allow all
```

## 规范要求
1. **没有明说，不得ci上传**：除非用户明确允许，否则不会执行CI上传操作
2. **没有明说，不得上传到git**：除非用户明确允许，否则不会执行git上传操作
3. **代码质量检查**：确保代码符合基本的质量要求
4. **安全性检查**：确保代码不包含敏感信息
5. **沙箱测试**：如果有沙箱测试的一律要运行，确保代码在沙箱环境中正常工作

## 示例场景
- 当用户修改代码后，需要检查是否可以上传
- 当用户需要控制代码上传权限时
- 当需要确保代码符合规范后再上传时

## 注意事项
- 此工具仅在明确授权的情况下执行上传操作
- 执行上传前会进行基本的代码检查
- 确保代码符合项目的编码规范
